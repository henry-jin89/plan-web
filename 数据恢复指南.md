# 🆘 数据恢复指南

## 问题说明

当您在浏览器设置中删除浏览数据时，会清除应用的所有本地数据，包括：
- ❌ 所有计划数据
- ❌ 同步配置信息
- ❌ 离线缓存数据
- ❌ 用户设置和偏好

这导致应用内容为空，同步功能失效。

## 🔧 立即恢复步骤

### 步骤1：检查自动备份
1. 打开应用主页
2. 点击 **🛡️ 数据备份** 卡片
3. 查看是否有可用的自动备份
4. 如有备份，点击"恢复此备份"

### 步骤2：紧急恢复（如果有备份）
1. 按 F12 打开浏览器开发者工具
2. 切换到"控制台"选项卡
3. 输入 `emergencyRestore()` 并按回车
4. 按照提示选择恢复备份

### 步骤3：重新配置同步
1. 打开 [同步设置页面](sync-settings.html)
2. 选择您之前使用的同步提供商：
   - GitHub 同步
   - Google Drive 同步
   - 自定义服务器
3. 重新登录授权
4. 点击"立即同步"从云端恢复数据

### 步骤4：手动恢复（如果有本地备份文件）
1. 如果您之前下载过备份文件
2. 打开备份管理器
3. 将备份文件内容复制到恢复区域
4. 点击恢复

## 🛡️ 预防措施

### 1. 启用自动备份
应用已自动启用以下保护机制：
- ✅ 每5分钟自动备份数据
- ✅ 多重备份存储（localStorage + IndexedDB）
- ✅ 同步配置独立备份
- ✅ 页面关闭前紧急备份

### 2. 定期下载备份
- 建议每周下载一次完整备份文件
- 备份文件可在任何设备上恢复

### 3. 浏览器设置优化
**重要：清除浏览数据时的正确操作**

清除浏览数据时，请按以下步骤操作：
1. 打开浏览器设置 → 隐私和安全 → 清除浏览数据
2. **取消勾选** "Cookie和其他网站数据" 
3. 或者在"高级"选项中排除此网站
4. 只清除缓存图像、下载记录等不重要数据

### 4. 云同步备份
- 确保同步功能正常工作
- 定期检查云端是否有最新数据
- 使用多个云服务提供商（推荐）

## 🔍 数据位置说明

### localStorage 数据键：
- `planData_day` - 日计划数据
- `planData_week` - 周计划数据
- `planData_month` - 月计划数据
- `sync_config` - 同步配置
- `backup_*` - 自动备份数据

### 查看数据的方法：
1. 按 F12 打开开发者工具
2. 切换到 "Application" 或 "存储" 选项卡
3. 展开 "Local Storage" 查看数据

## 📞 获取帮助

### 自助恢复工具
- 备份管理器：主页 → 数据备份
- 同步设置：[sync-settings.html](sync-settings.html)
- 紧急恢复：控制台输入 `emergencyRestore()`

### 常见问题
**Q: 为什么同步按钮显示"禁用同步"？**
A: 因为同步配置被清除，需要重新设置同步账户

**Q: 如何避免再次丢失数据？**
A: 
1. 不要清除"Cookie和网站数据"
2. 定期下载备份文件
3. 确保云同步正常工作

**Q: 数据能否完全恢复？**
A: 如果有以下任一备份，可以完全恢复：
- 自动备份（应用内）
- 云同步数据
- 手动下载的备份文件

## 💡 最佳实践

1. **定期备份**：每周下载一次完整备份
2. **多重保护**：同时使用本地备份和云同步
3. **正确清理**：清除浏览数据时排除此应用
4. **及时同步**：修改数据后立即同步到云端
