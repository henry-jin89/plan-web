## 🔬 工作原理

### 自动检测流程

1. **页面加载时**：
   ```
   智能启动器 → 检查现有数据 → 判断是否需要恢复
   ```

2. **如果需要恢复**：
   ```
   尝试多种恢复方法：
   ├── GitHub API直接恢复 (最强)
   ├── 通用仓库模式检测
   ├── 浏览器指纹匹配
   ├── 域名基础恢复
   └── 用户模式分析
   ```

3. **数据恢复过程**：
   ```
   找到数据源 → 下载数据 → 恢复到本地 → 配置同步 → 完成
   ```

**这是解决您问题的关键技术**：

1. **自动推测用户信息**：
   - 从域名提取用户名（如果是GitHub Pages）
   - 从localStorage残留信息推断
   - 从访问模式分析

2. **智能仓库检测**：
   - 尝试常见仓库名：`plan-data`、`personal-data`、`schedule-data`等
   - 无需Token即可检测公开仓库
   - 自动发现数据文件

3. **一键恢复**：
   - 找到数据后自动下载
   - 恢复所有计划和设置
   - 询问用户提供Token以启用同步
